# DESM

`DESM` — небольшой Python-модуль для численного решения уравнений движения спутника в декартовой системе координат.

Все вычисления выполняются в системе координат `ECI` (Earth-Centered Inertial): она инерциальна, фиксирована относительно звезд и в базовой постановке не учитывает вращение Земли.

Для решения практических задач рекомендуется использовать библиотеку `astropy`: при заданных начальных условиях она позволяет выполнять преобразования из `ECEF` (или другой системы координат) в `ECI` и обратно, чтобы восстанавливать траекторию относительно Земли.

Модуль поддерживает три режима динамики:
- центральное гравитационное поле Земли (задача о двух телах, `amendments='None'`);
- центральное поле + поправка `J2` (учет сплюснутости Земли первого порядка);
- центральное поле + поправки `J2` и `J3` (учет второй и третьей зональных гармоник).

При использовании режимов `amendments='J2'` и `amendments='J3'` влияние несферичности Земли проявляется в форме орбиты через поправки к геопотенциалу.

## Как подключить модуль в свой проект

Для использования `DESM` достаточно:
1. Скопировать файл `DESM.py` в папку вашего проекта.
2. Импортировать нужные функции/константы.

Пример импорта:

```python
from DESM import get_solution, R_e, mu
```

## Публичные константы

- `mu` (`float`) — гравитационный параметр Земли, м^3/с^2.
- `R_e` (`float`) — радиус Земли, м.
- `J2` (`float`) — коэффициент второй зональной гармоники гравитационного поля.
- `J3` (`float`) — коэффициент третьей зональной гармоники гравитационного поля.

## Основная функция

### `get_solution(position, time_interval, speed_proj, amendments='None') -> list`

Решает систему ОДУ и возвращает траекторию и скорости во времени.

#### Аргументы

- `position`: последовательность из 3 чисел (`list`, `tuple` или `numpy.ndarray`)
  - формат: `[x0, y0, z0]`
  - единицы: метры
- `speed_proj`: последовательность из 3 чисел (`list`, `tuple` или `numpy.ndarray`)
  - формат: `[vx0, vy0, vz0]`
  - единицы: м/с
- `time_interval`: последовательность из 2 чисел
  - формат: `[t0, t1]`
  - единицы: секунды
  - обязательное условие: `t1 > t0`
- `amendments`: `str`
  - `'None'` (по умолчанию) — без гармонических поправок
  - `'J2'` — с поправкой `J2`
  - `'J3'` — с поправками `J2` и `J3`

Рекомендуется передавать только значения `'None'`, `'J2'` или `'J3'`.

#### Возвращаемое значение

Список из 7 массивов `numpy` в следующем порядке:

```text
[x, y, z, vx, vy, vz, t]
```

где:
- `x, y, z` — координаты спутника (м),
- `vx, vy, vz` — проекции скорости (м/с),
- `t` — время (с).

Все массивы согласованы по длине: индекс `i` в каждом массиве относится к одному и тому же моменту `t[i]`.

#### Проверки входных данных

Перед расчетом функция валидирует входные параметры:
- `position` и `speed_proj` должны содержать ровно 3 конечных вещественных значения;
- `time_interval` должен содержать ровно 2 конечных вещественных значения;
- должно выполняться `t1 > t0`.

Если условия нарушены, выбрасываются исключения:
- `TypeError` — неверный тип контейнера/элементов;
- `ValueError` — неверная длина, нечисловые/бесконечные значения, некорректный интервал времени.

## Внутренние функции

В модуле также есть служебные функции с префиксом `_`:
- `_ODES(...)` — правая часть ОДУ для центрального поля;
- `_J2_ODES(...)` — правая часть ОДУ с поправкой `J2`;
- `_J3_ODES(...)` — правая часть ОДУ с поправками `J2` и `J3`;
- `_validate_vector(...)`, `_validate_time_interval(...)` — внутренние проверки аргументов.

Они предназначены для внутреннего использования, основной внешний API — `get_solution`.

## Пример использования

Готовый пример расчета и визуализации траектории есть в файле `TestModule.py`.

В нем показано:
- как задать начальные координаты и скорость;
- как вызвать `get_solution(...)`, включая режим `amendments='J3'`;
- как найти ближайшую точку к экваториальной плоскости (`Z=0`);
- как построить 3D-график орбиты и Земли через `matplotlib`;
- как задать одинаковые границы осей `X/Y/Z`, чтобы Земля визуально оставалась сферой.

## Минимальный пример

```python
import numpy as np
from DESM import get_solution, R_e, mu

position = [R_e + 400e3, 0.0, 0.0]
speed_proj = [0.0, np.sqrt(mu / (R_e + 400e3)), 0.0]
time_interval = [0.0, 5400.0]

x, y, z, vx, vy, vz, t = get_solution(
    position=position,
    time_interval=time_interval,
    speed_proj=speed_proj,
    amendments='None',
)

print(f"Points: {len(t)}")
print(f"Final position: {x[-1]:.1f}, {y[-1]:.1f}, {z[-1]:.1f}")
```
